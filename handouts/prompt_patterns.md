# AI 분석 의뢰 프롬프트 패턴 (배포용)

> AI로 통계 분석의 난이도를 낮추는 핵심은 **잘 시키는 것**. 좋은 결과는 좋은 프롬프트에서 나온다.
> 프롬프트엔 **데이터 구조 + 목표 + 도구 + 출력형식 + 검증요청**을 담는다.

---

## 1. 기본 골격

```
[역할] 너는 국제개발협력 데이터를 분석하는 통계 분석가야.
[데이터] OECD CRS 데이터. 컬럼: Year, DonorName, RecipientName, SectorName,
        FinanceType, USD_Disbursement, RecipientGDPpc, RecipientPop, LongDescription ...
        (예시 행 2~3개 붙여넣기)
[목표] 공여국별·분야별 집행액 교차표를 만들고, 분야 비중(%)도 보여줘.
[도구] pandas로 짜줘.   (또는: STATA .do 파일로)
[출력] 코드에 한국어 주석을 줄마다 달고, 결과 읽는 법을 2줄로 설명해줘.
[검증] 네가 임의로 가정한 것(결측 처리·단위 등)이 있으면 따로 표시해줘.
```

## 2. ★ 폐쇄망 실행용 (가장 중요)

폐쇄망에서 그대로 돌릴 STATA 코드를 받을 때:

```
이 STATA 코드를 추가 패키지(SSC) 없이 base STATA만으로, 인터넷 없이
오프라인에서 돌아가게 짜줘. STATA 버전 14에서도 되게 해줘.
경로·변수명은 파일 맨 위에 모아서 한 곳만 바꾸면 되게.
```

## 3. 도구 전환 (대조 학습)

```
방금 만든 pandas 분석을 STATA .do 파일로 똑같이 바꿔줘. 결과 숫자가 동일해야 해.
```

## 4. 검증 유도 (스스로 점검시키기)

```
이 결과가 틀릴 수 있는 지점 3가지를 스스로 점검해줘.
(결측, 단위, 이중계상, 명목/실질, 음수값 등)
```

## 5. 해석 요청 (의사결정 언어로)

```
이 회귀 결과를 통계 용어 없이, 국제개발협력 보고서에 쓸 2~3문장으로 해석해줘.
과장하지 말고, 단정할 수 없는 부분은 그렇게 표시해줘.
```

---

### 기억할 것
- AI는 **②코드 생성**을 맡는다. **①무엇을·왜 분석할지 설계**와 **③검증·해석**은 사람의 몫.
- "그럴듯해 보인다"와 "맞다"는 다르다 → 항상 [검증 체크리스트](verification_checklist.md)로 확인.
